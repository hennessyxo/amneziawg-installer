# awg-bot

[English](BOT.md) · [Русский](BOT.ru.md)

> **Telegram-бот** с контролем доступа для выдачи профилей AmneziaWG — Go, один бинарник.

![go](https://img.shields.io/badge/Go-1.25%2B-00ADD8?logo=go)
![telegram](https://img.shields.io/badge/Telegram-Bot%20API-26A5E4?logo=telegram)

Раздавай VPN-профили прямо из чата Telegram: пишешь `/new phone` — бот присылает
готовый `.conf` и QR. Работает на сервере рядом с VPN и использует то же ядро
`awgctl`, что и веб-панель.

## Возможности

- 💬 **Команды**: `/new <имя>` (создаёт клиента, шлёт `.conf` + QR), `/list`,
  `/config <имя>` (прислать снова), `/revoke <имя>`, а также `/start` и `/help`.
- 🔐 **Контроль доступа (два фактора)**: пользователь допускается, только если он
  **одновременно** в **списке разрешённых** (Telegram ID) **и** ввёл **пароль доступа**
  через `/auth <пароль>`. Пользователь не из списка не пройдёт даже с верным паролем;
  удаление ID из списка сразу отзывает доступ. Факт ввода пароля запоминается между
  перезапусками.
- 📡 **Без входящего порта**: бот сам опрашивает Telegram (**long-poll**), поэтому на
  сервере ничего открывать не нужно — работает за любым фаерволом/NAT.
- 📦 **Один бинарник**: никаких лишних служб; переиспользует конфиг и каталог клиентов установщика.

## Получить токен бота

1. Открой [@BotFather](https://t.me/BotFather) в Telegram → `/newbot` → следуй
   подсказкам. Он выдаст **токен** вида `123456:ABC-DEF...`.
2. (Опционально) Чтобы заранее задать админов, узнай свой числовой Telegram ID через
   [@userinfobot](https://t.me/userinfobot).

## Установка

Через меню установщика (рекомендуется):

```bash
sudo bash amneziawg-install.sh   # → пункт 8 «Telegram-бот»
```

Скачает бинарник, спросит **токен** и **пароль доступа**, затем запустит
systemd-службу. Неинтерактивно (автоматизация / SSH):

```bash
AWG_BOT_TOKEN='123456:ABC...' \
AWG_BOT_ADMINS='12345678,87654321' \
AWG_BOT_PASSWORD='Admin2@' \
  sudo -E bash amneziawg-install.sh --install-bot
sudo bash amneziawg-install.sh --remove-bot
```

Нужны **и** список (`AWG_BOT_ADMINS`), **и** пароль. Затем из разрешённого аккаунта в
Telegram: один раз `/auth <пароль>`, потом `/new laptop`.

## Флаги

| Флаг | По умолчанию | Значение |
|------|--------------|----------|
| `--token-file` | — | файл с токеном бота (предпочтительнее `--token`) |
| `--admins` | — | Telegram ID через запятую, которым всегда разрешено |
| `--password-hash-file` | — | bcrypt-хеш пароля доступа (`awg-bot hash`) |
| `--auth-store` | `/etc/amnezia/amneziawg/bot-authorized.json` | запомненные авторизованные чаты |
| `--iface` | `awg0` | интерфейс AmneziaWG |
| `--conf` / `--params` / `--client-dir` / `--store` | пути установщика | конфиг сервера + данные клиентов |
| `--lang` | `ru` | язык ответов бота (`ru`/`en`) |

`awg-bot hash` читает пароль из stdin и печатает bcrypt-хеш (открытый пароль нигде
не хранится). Нужны **оба**: `--admins` и `--password-hash-file` — пользователь должен
быть в списке *и* ввести пароль.

## Безопасность

- **Токен** — секрет: хранится `600` под `umask 077`, никогда не попадает в репозиторий.
- Доступ **двухфакторный**: список (кто) **и** пароль (подтверждение). Бот отвечает на
  команды управления, только когда выполнены оба условия; остальным — «нет доступа».
  Задай надёжный пароль (то же правило, что у панели: строчные и ЗАГЛАВНЫЕ буквы,
  цифра и спецсимвол).
- Когда пользователь шлёт `/auth <пароль>`, бот **удаляет это сообщение**, чтобы пароль
  не оставался в истории чата.
- Бот работает от root (нужно для `awg`/`awg-quick`), как и панель.

## Архитектура

```
cmd/awg-bot/main.go      # флаги, связка, подкоманда `hash`
internal/tgbot/
├── api.go               # минимальный клиент Telegram Bot API (long-poll, multipart)
├── auth.go              # ID админов + bcrypt-пароль + сохранённые авторизованные чаты
├── command.go           # чистый парсинг команд
├── bot.go               # цикл опроса + обработка команд
└── i18n.go              # ответы RU/EN
```

Переиспользует `internal/awgctl` (add/remove/list/config) и `internal/auth` (bcrypt).
Чистая логика (парсинг команд, авторизация) покрыта тестами `go test ./...`.
