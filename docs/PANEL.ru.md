# awg-panel

[English](PANEL.md) · [Русский](PANEL.ru.md)

> Веб-панель управления VPN на **AmneziaWG** — Go + htmx, один бинарник.

![go](https://img.shields.io/badge/Go-1.25%2B-00ADD8?logo=go)
![ui](https://img.shields.io/badge/UI-htmx-3366cc)

Дашборд с авторизацией по сессии: смотреть живой трафик клиентов и управлять ими
в браузере. Построен на том же ядре разбора `awg`, что и `awg-monitor`.

## Возможности

- 🔐 **Авторизация**: пароль администратора (bcrypt), сессии в HttpOnly-куках, CSRF на формах.
- 🔒 **HTTPS**: работает по TLS (самоподписанный сертификат ставится автоматически).
- 📊 **Живой дашборд**: онлайн-статус, скорость ↑↓, суммарный трафик по клиентам (htmx-поллинг).
- 📅 **Трафик по времени**: расход на клиента за последние **день / неделю / месяц**,
  с **сортировкой** по столбцам (клик по заголовку) — сразу видно, кто сколько потратил.
- 🖥️ **Обзор сервера** (страница *Сервер*): нагрузка хоста — CPU %, load average,
  память и диск, аптайм — плюс сводный трафик клиентов за день / неделю / месяц, итог
  за всё время, график трафика за 30 дней и топ-клиенты. Читает `/proc` напрямую.
- ➕ **Управление**: добавить / удалить / отключить / включить / **переименовать** клиента, скачать `.conf`, QR.
- ✏️ **Редактирование на лету**: у существующего клиента можно изменить скорость, квоту и срок
  (кнопка «изм.») — без пересоздания.
- ⏳ **Квоты и срок действия**: при создании клиента задаёшь лимит трафика (ГБ) и/или срок (дней);
  фоновый enforcer считает трафик (с учётом сброса счётчиков при перезапуске) и сам
  **отключает** истёкших и превысивших квоту клиентов — они сохраняются и их можно
  **включить обратно** (при включении истёкший срок снимается, переполненная квота обнуляется).
- 🐢 **Ограничение скорости**: задаёшь лимит в Мбит/с — фоновый шейпер на `tc` режет отдачу
  (HTB-класс на IP клиента) и приём (ingress-police) вместо отключения.
- 📦 **Один бинарник**: HTML/CSS/htmx вшиты — деплоить отдельно нечего.

## Установка

Через меню установщика (рекомендуется):

```bash
sudo bash amneziawg-install.sh   # → пункт 7 «Веб-панель»
```

Установщик скачает бинарник, спросит пароль администратора, сгенерирует
самоподписанный TLS-сертификат и поднимет systemd-службу на `https://<ip>:8443`.

Вручную из исходников:

```bash
go build -o awg-panel ./cmd/awg-panel
echo 'мой-пароль' | ./awg-panel hash > /etc/amnezia/amneziawg/panel.hash
sudo ./awg-panel \
  --password-hash-file /etc/amnezia/amneziawg/panel.hash \
  --tls-cert cert.pem --tls-key key.pem
```

## Флаги

| Флаг | По умолчанию | Назначение |
|------|--------------|-----------|
| `--listen` | `:8443` | адрес прослушивания |
| `--iface` | `awg0` | интерфейс AmneziaWG |
| `--conf` | `/etc/amnezia/amneziawg/awg0.conf` | конфиг сервера |
| `--params` | `/etc/amnezia/amneziawg/params` | параметры установщика |
| `--client-dir` | `/etc/amnezia/amneziawg/clients` | где хранятся конфиги, созданные панелью |
| `--store` | `/etc/amnezia/amneziawg/clients.json` | метаданные жизненного цикла (квоты/срок) |
| `--password-hash-file` | `/etc/amnezia/amneziawg/panel.hash` | bcrypt-хеш пароля админа |
| `--tls-cert` / `--tls-key` | — | включают HTTPS |

`awg-panel hash` читает пароль из stdin и печатает bcrypt-хеш (открытый пароль нигде не хранится).

Чтобы сменить пароль администратора позже без переустановки, перезапишите файл хеша
и перезапустите службу:

```bash
echo 'новый-пароль' | awg-panel hash > /etc/amnezia/amneziawg/panel.hash
systemctl restart awg-panel
```

Вкладка **«Настройки»** в десктоп-приложении делает ровно это по SSH.

## Безопасность

- Панель запускается под root (нужно для `awg`/`awg-quick`).
- **Защита от подбора пароля:** после 5 неверных попыток вход с этого IP блокируется
  на 15 минут. Пароль — bcrypt, с проверкой сложности (строчные и заглавные буквы,
  цифра и спецсимвол), которую делает установщик.
- Куки `HttpOnly` + `SameSite=Lax`, флаг `Secure` ставится при HTTPS.
- На формах — CSRF-токен, привязанный к сессии.
- **Максимальная безопасность (если параноишь):** не открывай порт `8443` наружу,
  а ходи в панель через SSH-туннель:
  ```bash
  ssh -L 8443:localhost:8443 root@SERVER   # затем открой https://localhost:8443
  ```
  Тогда панель недоступна из интернета вообще, даже зная пароль.
- Клиенты, созданные при установке или через CLI, **подхватываются панелью**
  автоматически (можно менять лимиты, отключать, переименовывать). Их `.conf`
  зеркалируется в каталог панели, так что скачивание/QR тоже работают.

## Архитектура

```
cmd/awg-panel/main.go        # флаги, TLS, подкоманда `hash`
internal/
├── awgctl/                  # control plane (params, добавление/удаление пиров, FileController)
├── auth/                    # bcrypt + сессии в памяти + CSRF
├── lifecycle/               # хранилище квот/срока, учёт трафика, дневные снимки, движок правил
├── shaper/                  # планировщик команд tc (лимиты скорости на клиента)
├── sysstat/                 # сбор нагрузки хоста (CPU/RAM/диск/аптайм из /proc)
├── server/                  # роутинг, middleware, хендлеры, учёт скорости, enforcer
└── web/                     # встроенные шаблоны + статика (htmx, CSS)
```

Enforcer (в `server`) сверяется каждые 30 с: учитывает трафик в хранилище
`lifecycle`, затем отключает превысивших квоту и истёкших клиентов. Лимиты скорости
переприменяются через `shaper` при каждом изменении и при старте.

Чистая логика (`awgctl`, `auth`) и HTTP-хендлеры (против фейкового `Controller`)
покрыты юнит-тестами; запуск: `go test ./...`.
