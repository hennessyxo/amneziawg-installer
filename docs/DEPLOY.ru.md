# awg-deploy

[English](DEPLOY.md) · [Русский](DEPLOY.ru.md)

> Кросс-платформенный инструмент: устанавливает и управляет AmneziaWG на удалённом
> сервере по SSH прямо с твоего компьютера (Windows `.exe`, macOS, Linux). Один
> бинарник, на сервере заранее ничего ставить не нужно.

![go](https://img.shields.io/badge/Go-1.25%2B-00ADD8?logo=go)
![platform](https://img.shields.io/badge/platform-Windows%20%7C%20macOS%20%7C%20Linux-orange)

`awg-deploy` встраивает скрипт установщика, передаёт его на сервер по SSH, запускает
неинтерактивно, забирает конфиг клиента, сохраняет его как `.conf` и открывает
сканируемую QR-картинку. Не нужно заходить по SSH вручную или знать Linux.

## Скачать

Возьми бинарник под свою ОС в [Releases](https://github.com/hennessyxo/awg-suite/releases):

| Твой компьютер | Файл |
|----------------|------|
| Windows | `awg-deploy-windows-amd64.exe` |
| macOS, Apple Silicon (M1–M5) | `awg-deploy-darwin-arm64.tar.gz` |
| macOS, Intel | `awg-deploy-darwin-amd64.tar.gz` |
| Linux, x86_64 | `awg-deploy-linux-amd64.tar.gz` |
| Linux, ARM | `awg-deploy-linux-arm64.tar.gz` |

> `darwin` = macOS. Качай под **свой** компьютер, а не под сервер. Сервер
> настраивается автоматически по SSH. Архивы сохраняют флаг «исполняемый»
> (`chmod` не нужен).

### macOS: Gatekeeper

Распакуй `.tar.gz` (двойной клик или `tar -xzf …`). Бинарник неподписанный,
поэтому сними карантин один раз:

```bash
xattr -dr com.apple.quarantine ./awg-deploy
./awg-deploy
```

Либо: правый клик по файлу в Finder → **Открыть** → **Открыть** (или
`Системные настройки → Конфиденциальность и безопасность → «Открыть всё равно»`).

## Мастер (самый простой путь)

Запусти бинарник **без аргументов** (на Windows: двойной клик по `.exe`):

```bash
./awg-deploy-darwin-arm64
```

Спросит адрес сервера и пароль root, подключится по SSH и запустит установку и меню
управления **прямо на сервере** (через интерактивную SSH-сессию). Все действия
(установка, добавление/удаление/переименование клиентов, мониторинг, веб-панель)
выполняются на сервере; этот инструмент просто «заходит» туда за тебя. Команды ниже
нужны тем, кто хочет вызывать действия напрямую (скрипты/автоматизация).

## Использование

```bash
# Установить VPN на сервер (спросит SSH-пароль, если не указан ключ):
awg-deploy install root@IP_СЕРВЕРА --client phone

# С SSH-ключом и нестандартным SSH-портом:
awg-deploy install root@IP_СЕРВЕРА:2222 --identity ~/.ssh/id_ed25519

# Добавить ещё клиента (печатает его конфиг + QR):
awg-deploy add-client root@IP_СЕРВЕРА laptop

# Список клиентов:
awg-deploy list root@IP_СЕРВЕРА

# Удалить клиента:
awg-deploy remove-client root@IP_СЕРВЕРА laptop

# Интерактивное меню сервера прямо в твоём терминале (по SSH):
awg-deploy menu root@IP_СЕРВЕРА

# Живой мониторинг сервера из своего терминала:
awg-deploy monitor root@IP_СЕРВЕРА

# Полностью удалить AmneziaWG с сервера (спросит подтверждение):
awg-deploy uninstall root@IP_СЕРВЕРА
```

> Повторный `install` на уже настроенном сервере ничего не ломает: он распознаёт
> это и подсказывает команды управления (`add-client`, `list`, `remove-client`,
> `monitor`). Интерактивное меню есть и на самом сервере: `sudo bash amneziawg-install.sh`.

На Windows просто запусти `.exe` из терминала (PowerShell/Windows Terminal):

```powershell
.\awg-deploy-windows-amd64.exe install root@IP_СЕРВЕРА
```

### Флаги install

| Флаг | Назначение |
|------|-----------|
| `--preset` | пресет обфускации; по умолчанию `mobile` (MTU 1280, Jc=3, работает и на мобильном, и на ПК) |
| `--port` | UDP-порт AmneziaWG (по умолчанию свободный случайный) |
| `--client` | имя первого клиента |
| `--server-ip` | публичный IP/домен для клиентов (по умолчанию автоопределение) |
| `--dns1`, `--dns2` | DNS клиентов |
| `--out` | куда сохранить `.conf` локально |
| `--identity` | SSH-приватный ключ (иначе спросит пароль) |
| `--known-hosts` | путь к known_hosts |
| `--accept-new` | довериться новому хосту без вопроса |

## Безопасность

- Ключ хоста проверяется по `known_hosts`. Неизвестный хост → запрос подтверждения
  (TOFU) с показом отпечатка SHA256; **изменившийся** ключ → отказ (возможный MITM).
- Пароль читается скрытым вводом и не сохраняется.
- Скрипт установщика **встроен в бинарник**, отдельно ничего качать не нужно.

## Как это работает

```
awg-deploy ──SSH──> сервер
   │  передаёт встроенный amneziawg-install.sh в `bash -s -- --yes`
   │  настройки через переменные AWG_* (неинтерактивный режим)
   │  забирает конфиг клиента из stdout
   └─ сохраняет <имя>.conf + <имя>.png (QR) локально и открывает картинку
```

`monitor` на каждом тике выполняет `awg show <iface> dump` по SSH и рисует тот же
TUI, что и [`awg-monitor`](MONITOR.ru.md), переиспользуя `internal/awg` и `internal/ui`.
