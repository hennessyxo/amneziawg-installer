# Экспертный режим

[English](EXPERT.md) · [Русский](EXPERT.ru.md)

> Для обычных пользователей всё работает без настроек. Если ты понимаешь, что
> делаешь, каждая крутилка ниже **опциональна** и по умолчанию равна безопасным
> значениям, так что обычная установка не меняется.

Экспертные настройки бывают двух видов:

1. **Серверные**: задаются один раз при установке (профиль/параметры обфускации, порт, DNS).
2. **На клиента**: задаются при добавлении клиента (split-tunnel, DNS, MTU).

---

## 1. Профиль обфускации (при установке)

AmneziaWG прячет рукопожатие WireGuard за мусорными пакетами и кастомными
заголовками. Профиль выбирается при установке:

| Профиль | Что делает |
|---------|------------|
| `mobile` | **По умолчанию.** MTU 1280, Jc=3, умеренный мусор. Стабильно и на 4G/LTE, и на ПК. |
| `desktop` | Выше MTU (1420), чуть больше мусора. Для проводных/ПК-каналов. |
| `plain` | **Чистый WireGuard**: мусор/заголовки выключены, совместим со стандартным WG. |
| `custom` | Параметры задаёшь сам (см. ниже). |

**Где выбрать:**
- **Десктоп-приложение** → экран установки → *Дополнительно* → *Профиль обфускации*.
- **Установщик (интерактивно / `awg-deploy`)** → спросит при установке (Enter = mobile).
- **Неинтерактивно / скрипты** → `AWG_PRESET=plain sudo -E bash amneziawg-install.sh --yes`.

### Отдельные параметры (env, при установке)

```bash
AWG_PRESET=custom \
AWG_JC=2 AWG_JMIN=40 AWG_JMAX=90 \
AWG_S1=100 AWG_S2=140 \
AWG_H1=… AWG_H2=… AWG_H3=… AWG_H4=… \
AWG_MTU=1380 \
sudo -E bash amneziawg-install.sh --yes
```

Любое незаданное значение остаётся дефолтом профиля (для `Jmin/Jmax/S1/S2/H1-H4` случайным).

> ⚠️ **`S1`, `S2`, `H1`–`H4` обязаны совпадать на сервере и каждом клиенте.** Они
> зашиваются в `.conf` каждого клиента при создании, поэтому если поменять их потом,
> придётся **перевыпустить все конфиги клиентов**. `Jc/Jmin/Jmax` могут отличаться у пиров.

## 2. Порт и DNS (при установке)

- **Порт**: `AWG_PORT=51820` (пусто = подбирается свободный случайный UDP-порт). Также в
  *Дополнительно* десктоп-приложения и в вопросе установщика.
- **DNS**: `AWG_DNS1=1.1.1.1 AWG_DNS2=1.0.0.1` (серверный дефолт для новых клиентов). Также
  в *Дополнительно* приложения и в вопросе установщика.

## 3. Настройки на клиента (при добавлении)

Каждый клиент может переопределить дефолты. Это удобно для **split-tunnel** (гнать через
VPN только нужные подсети), своего резолвера или другого MTU.

| Поле | По умолчанию | Пример |
|------|--------------|--------|
| **Маршруты** (`AllowedIPs`) | `0.0.0.0/0,::/0` (весь трафик) | `10.0.0.0/8,192.168.0.0/16` (split) |
| **DNS** | серверный дефолт | `9.9.9.9` |
| **MTU** | серверный дефолт | `1380` |

**Где задать:**
- **Веб-панель** → *Добавить клиента* → *Расширенные*.
- **Десктоп-приложение** → вкладка Клиенты → *Расширенные настройки клиента*.
- **Установщик / `--add-client`** (env):
  ```bash
  AWG_ALLOWED_IPS='10.0.0.0/8,192.168.0.0/16' AWG_CLIENT_DNS='9.9.9.9' \
    sudo -E bash amneziawg-install.sh --add-client work-laptop
  ```

Маршруты (`AllowedIPs`) меняют только то, что **этот клиент** отправляет в туннель. На
серверную маршрутизацию и адресацию они не влияют, поэтому их безопасно крутить на клиента.

## Примеры

**Сервер на чистом WireGuard** (без обфускации, напр. для доверенной сети):
```bash
AWG_PRESET=plain sudo -E bash amneziawg-install.sh --yes
```

**Клиент, который гонит через VPN только корпоративную подсеть** (остальное напрямую):
```bash
AWG_ALLOWED_IPS='10.20.0.0/16' sudo -E bash amneziawg-install.sh --add-client office
```

## Предостережения

- `S1/S2/H1-H4` должны совпадать сервер ↔ клиент (см. предупреждение выше).
- Режим `plain` убирает обфускацию против DPI, так что используй только там, где цензура не проблема.
- Некорректные значения на клиента игнорируются (берётся безопасный дефолт), а не пишутся в конфиг.

## Пока не настраивается

- **Своя подсеть VPN** (`10.66.66.0/24`): в планах. Она пронизывает выдачу IP клиентам,
  поэтому требует аккуратной протестированной работы, прежде чем её открывать.
